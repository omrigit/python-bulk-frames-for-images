Tech Stack
==========

Languages
---------
- Python 3.10+

Frameworks / Runtime
--------------------
- Standard Python CLI (`argparse`)

UI / Interface
--------------
- Command-line interface (MVP)
- Optional Web UI can be added later using the same processing core

Image Processing
----------------
- Pillow (`PIL`) for image IO, framing, EXIF normalization, and text rendering

Data Layer
----------
- File-system based input/output folders
- No database in MVP

Project Architecture
--------------------
- `src/framebulk/cli.py`:
  Parses CLI args and executes `apply`, `list-fonts`, and `color-cheatsheet` commands.
- `src/framebulk/processor.py`:
  Traverses input directory, filters image files, handles output naming, and summarizes results.
- `src/framebulk/image_ops.py`:
  Core image transformation logic: EXIF fix, orientation-aware frame/text placement, and rendering.
- `src/framebulk/fonts.py`:
  Font lookup and loading strategy with fallbacks.
- `src/framebulk/models.py`:
  Shared config/result dataclasses.
- `tests/`:
  Unit tests for orientation behavior, text fitting, and batch filtering.

Key Dependencies
----------------
- Pillow >= 10.0.0
- pytest >= 8.0.0 (dev/testing)

How To Run
----------
1. Create and activate virtual environment:
   - `python -m venv .venv`
   - `source .venv/bin/activate`
2. Install:
   - `pip install -e ".[dev]"`
3. Run CLI:
   - `framebulk apply --input <dir> --output <dir> --frame-color white --margin-top 60 --margin-right 60 --margin-bottom 140 --margin-left 60 --text "Caption" --font-size 42 --text-position auto --text-align center --jpeg-quality 90 --preserve-metadata`
   - `framebulk color-cheatsheet --output ./out/color-cheatsheet.jpg`
4. Run tests:
   - `pytest`
